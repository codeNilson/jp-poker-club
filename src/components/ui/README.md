# ui

Componentes base de interface.

Aqui devem ficar os componentes mais atômicos do projeto, como botão, input, dialog, dropdown, sheet, toast e outros blocos do shadcn/ui ou equivalentes.

Esses componentes devem ser o mais reutilizáveis possível e não devem conter regras de negócio.

Componentes atuais:
- `alert.tsx`: componente de alerta (variantes padrão e destrutiva).
- `button.tsx`: botão base com variantes e tamanhos.
- `dropdown-menu.tsx`: menu suspenso baseado em Radix UI.
- `skeleton.tsx`: placeholder de carregamento animado.
- `sonner.tsx`: wrapper do toast Sonner configurado para o tema do projeto.
