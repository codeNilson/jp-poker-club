# forms

Componentes ligados a formulários.

Aqui entram campos especializados, máscaras, grupos de inputs, selects compostos, validações visuais e elementos que encapsulam interação de formulário.

Se um campo depender muito de uma regra de negócio específica, vale mover a lógica para `services` ou `lib` e manter este diretório focado na UI do formulário.
